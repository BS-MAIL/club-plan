# GeoEquation Lab 개발 로드맵

> **목표**: 수업 적용 가능한 최소 기능 제품(MVP)을 빠르게 만들고, 실제 수업 데이터를 바탕으로 개선한다.

---

## 1. 전체 일정 개요

```text
1단계: 기획 정리        → 수업 목표, 기능 범위, 미션 설계
2단계: MVP 개발         → 좌표평면, 직선 실험실, 원 실험실 구현
3단계: 수업 자료 제작   → 활동지, 교사용 가이드, 설문지 제작
4단계: 파일럿 테스트    → 소규모 학생 또는 동료 교사 테스트
5단계: 현장 수업 적용   → 2차시 수업 수행 및 자료 수집
6단계: 분석·개선        → 로그·설문 분석, 앱과 수업안 개선
```

---

## 2. 단계별 계획

### 1단계. 기획 정리
| 항목 | 내용 | 산출물 |
|:---|:---|:---|
| 교육 목표 확정 | 직선의 방정식과 원의 방정식으로 범위 제한 | 프로젝트 기획서 |
| 미션 설계 | 직선·원 미션 8개 내외 작성 | 미션 목록 |
| 화면 흐름 설계 | 홈, 실험실, 교사용 보기 구성 | 기능명세서 |
| 평가 계획 수립 | 루브릭, 설문, 로그 항목 결정 | 수업지도안 |

### 2단계. MVP 개발
| 기능 | 우선순위 | 완료 기준 |
|:---|:---:|:---|
| 좌표평면 렌더링 | 높음 | 축, 격자, 좌표 표시 가능 |
| 점 드래그 | 높음 | 점 이동 시 좌표 갱신 |
| 직선 표시 | 높음 | 두 점을 지나는 직선과 기울기 표시 |
| 원 표시 | 높음 | 중심, 반지름 조작 가능 |
| 식 입력 | 중간 | 기본 직선·원 식 입력 가능 |
| 미션 카드 | 중간 | 미션 안내와 완료 버튼 제공 |
| 로그 저장 | 중간 | JSON/CSV 다운로드 가능 |
| 교사용 대시보드 | 낮음 | 간단한 완료 현황 표시 |

### 3단계. 수업 자료 제작
| 자료 | 내용 |
|:---|:---|
| 학생 활동지 | 예측, 관찰, 일반화 문항 포함 |
| 교사용 진행안 | 차시별 시간 운영, 발문, 예상 오류 포함 |
| 설문지 | 흥미도, 난이도, 도움 정도, 개선 의견 |
| 증거자료 수집 양식 | 사진, 스크린샷, 학생 결과물 목록 |

### 4단계. 파일럿 테스트
| 점검 항목 | 확인 질문 |
|:---|:---|
| 사용성 | 학생이 설명 없이도 점을 움직일 수 있는가? |
| 수학 정확성 | 계산값과 그래프가 정확한가? |
| 난이도 | 미션이 너무 쉽거나 어렵지 않은가? |
| 수업 흐름 | 50분 안에 활동과 정리가 가능한가? |
| 오류 처리 | 잘못된 입력에 안내가 충분한가? |

### 5단계. 현장 수업 적용
| 차시 | 주요 활동 | 수집 자료 |
|:---:|:---|:---|
| 1차시 | 직선의 방정식 탐구 | 활동지, 로그, 사진 |
| 2차시 | 원의 방정식 탐구 | 활동지, 로그, 설문, 발표 자료 |

### 6단계. 분석·개선
| 분석 내용 | 개선 방향 |
|:---|:---|
| 체류 시간이 긴 미션 | 설명 보강 또는 힌트 추가 |
| 오답률 높은 식 입력 | 입력 UI 개선, 예시 추가 |
| 흥미도 낮은 활동 | 게임형 미션 또는 협업 요소 추가 |
| 개념 정리 부족 | 탐구 후 일반화 질문 강화 |

## 3. MVP 개발 체크리스트

### 수학 기능
- [ ] 좌표축과 격자가 정확하게 표시된다.
- [ ] 점을 드래그하면 좌표가 갱신된다.
- [ ] 두 점을 지나는 직선의 기울기가 표시된다.
- [ ] 직선 방정식이 올바르게 표시된다.
- [ ] 원의 중심과 반지름이 식과 연결되어 표시된다.
- [ ] 식 입력 결과가 그래프와 비교된다.

### 수업 기능
- [ ] 미션 카드가 단계별로 제공된다.
- [ ] 학생이 힌트를 요청할 수 있다.
- [ ] 탐구 기록을 작성할 수 있다.
- [ ] 결과를 스크린샷 또는 텍스트로 제출할 수 있다.

### 데이터 기능
- [ ] 익명 세션 ID가 생성된다.
- [ ] 미션 시작·제출·완료 이벤트가 기록된다.
- [ ] 힌트 사용 횟수가 기록된다.
- [ ] 로그를 CSV 또는 JSON으로 다운로드할 수 있다.

## 4. 권장 기술 방향

> 실제 구현 단계에서 저장소 상황과 배포 환경에 맞춰 조정한다.

| 영역 | 권장 방향 | 이유 |
|:---|:---|:---|
| 프론트엔드 | HTML/CSS/JavaScript 또는 React | 브라우저 접근성, 빠른 개발 |
| 그래프 렌더링 | SVG 또는 Canvas | 좌표평면·도형 조작에 적합 |
| 수식 처리 | 직접 파서 최소 구현 또는 제한된 입력 템플릿 | MVP 안정성 확보 |
| 데이터 저장 | LocalStorage + CSV 다운로드 | 개인정보 부담 감소, 간단한 수업 적용 |
| 배포 | GitHub Pages, Vercel, Netlify 등 | 링크 기반 공유 용이 |
