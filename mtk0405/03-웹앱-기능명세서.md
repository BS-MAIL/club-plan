# GeoEquation Lab 웹앱 기능명세서

> **목표**: 직선의 방정식과 원의 방정식 이해를 위한 브라우저 기반 인터랙티브 실습 앱의 기능, 화면, 데이터 구조를 정의한다.

---

## 1. MVP 범위

초기 버전은 개발 부담을 줄이고 수업 적용 가능성을 높이기 위해 아래 기능에 집중한다.

### 포함 기능
- 좌표평면 캔버스
- 두 점을 지나는 직선 표시
- 기울기·절편 표시
- 중심·반지름 조작을 통한 원 표시
- 식 입력 후 그래프와 비교
- 미션 카드와 힌트
- 익명 학습 로그 저장 또는 다운로드

### 제외 또는 후순위 기능
- 회원가입·로그인
- 복잡한 CAS 수준의 수식 해석
- 실시간 교사-학생 동기화
- 포물선·타원·쌍곡선 등 다른 도형
- 궤적 탐구, 점과 직선 사이 거리 등 확장 개념
- 자동 채점 고도화

---

## 2. 주요 화면 구조

```text
[홈]
 ├─ 수업 코드 입력 / 익명 닉네임 입력
 ├─ 학습 모드 선택
 │   ├─ 직선 실험실
 │   └─ 원 실험실
 └─ 교사용 보기

[실험실 화면]
 ├─ 좌측: 미션 카드 / 힌트 / 탐구 질문
 ├─ 중앙: 좌표평면 캔버스
 ├─ 우측: 식 입력 / 계수 슬라이더 / 현재 값 패널
 └─ 하단: 탐구 기록 / 제출 버튼

[교사용 보기]
 ├─ 미션별 완료 현황
 ├─ 평균 체류 시간
 ├─ 힌트 사용 횟수
 └─ CSV 다운로드
```

## 3. 화면별 기능

### 3.1 홈 화면
| 기능 | 설명 |
|:---|:---|
| 수업 코드 입력 | 교사가 제시한 코드 입력. MVP에서는 임의 문자열 허용 |
| 익명 닉네임 | 예: `A01`, `B07`, `팀3` |
| 모드 선택 | 직선 실험실, 원 실험실 중 선택 |
| 개인정보 안내 | 이름·학번 입력 금지, 익명 수집 안내 |

### 3.2 좌표평면 캔버스
| 기능 | 설명 |
|:---|:---|
| 격자 표시 | x축, y축, 눈금 표시 |
| 확대·축소 | 필요 시 마우스 휠 또는 버튼으로 조작 |
| 점 드래그 | 점 A, B, C 등을 이동 |
| 스냅 기능 | 정수 좌표에 맞추기 on/off |
| 좌표 표시 | 점 옆에 `(x, y)` 표시 |

### 3.3 직선 실험실
| 기능 | 설명 |
|:---|:---|
| 기울기 슬라이더 | `a` 값을 바꾸며 `y = ax + b`의 기울기 변화 확인 |
| y절편 슬라이더 | `b` 값을 바꾸며 직선의 평행이동 확인 |
| 두 점을 지나는 직선 | A, B를 지나는 직선 자동 표시 |
| 기울기 계산 | `m = (y₂-y₁)/(x₂-x₁)` 표시 |
| 직선 방정식 표시 | `y = mx + b` 또는 `ax + by + c = 0` |
| 조건 미션 | 기울기와 y절편, 한 점과 기울기, 두 점 조건 |
| 식 입력 비교 | 학생 입력식과 목표 직선 비교 |

### 3.4 원 실험실
| 기능 | 설명 |
|:---|:---|
| 중심 드래그 | 원의 중심 C 이동 |
| 반지름 슬라이더 | 반지름 r 조절 |
| 표준형 표시 | `(x-a)^2 + (y-b)^2 = r^2` |
| 표준형 집중 | `(x-a)^2 + (y-b)^2 = r^2` 형태만 다룸 |
| 조건 미션 | “중심이 (2,-1), 반지름 3인 원의 식을 입력하라.” |

## 4. 미션 예시

### 직선 미션
1. 점 (1,2)를 지나고 기울기가 2인 직선을 만들어라.
2. 두 점 A(-1,3), B(3,-1)을 지나는 직선의 방정식을 입력하라.
3. `y = 2x + b`에서 `b`를 바꾸면 직선은 어떻게 변하는가?
4. 서로 평행한 직선들의 공통점은 무엇인가?

### 원 미션
1. 중심이 (2,3), 반지름이 4인 원을 만들고 식을 입력하라.
2. `(x+1)^2 + (y-2)^2 = 9`의 중심과 반지름을 말하라.
3. 반지름이 같고 중심만 다른 원들은 어떻게 이동하는가?
4. 중심이 원점인 원과 그렇지 않은 원의 식은 무엇이 다른가?

## 5. 피드백 메시지 설계

| 상황 | 메시지 예시 |
|:---|:---|
| 정답 | “좋아요. 입력한 식이 그래프와 일치합니다.” |
| 부호 오류 | “중심 좌표의 부호를 다시 확인해 보세요. `(x-a)`는 x좌표가 a인 중심을 의미합니다.” |
| 기울기 오류 | “두 점의 y좌표 차이와 x좌표 차이를 다시 비교해 보세요.” |
| 반지름 오류 | “오른쪽 값은 반지름이 아니라 반지름의 제곱입니다.” |
| 입력 형식 오류 | “예시처럼 입력해 보세요: y=2x+1 또는 (x-1)^2+(y+2)^2=9” |

## 6. 데이터 로깅 명세

### 6.1 로그 항목
| 필드 | 예시 | 설명 |
|:---|:---|:---|
| sessionId | `2026-04-05-A01` | 익명 세션 ID |
| missionId | `line-02` | 미션 ID |
| eventType | `start`, `submit`, `hint`, `complete` | 이벤트 유형 |
| timestamp | `2026-04-05T10:15:00` | 발생 시각 |
| value | `{m:2,b:1}` | 입력값 또는 조작값 |
| result | `correct`, `wrong`, `retry` | 결과 |
| elapsedSec | `132` | 미션 체류 시간 |

### 6.2 개인정보 원칙
- 학생 이름, 학번, 전화번호, 이메일 수집 금지
- 익명 코드만 사용
- 수업 후 분석이 끝나면 원자료 삭제 또는 비식별 보관
- 사진·영상은 별도 동의서와 모자이크 처리 원칙 준수

## 7. 교사용 대시보드 초안

| 영역 | 표시 내용 |
|:---|:---|

## 8. UI 원칙

1. **수학이 먼저 보이게 한다**: 장식보다 좌표평면과 식의 가독성을 우선한다.
2. **조작 결과가 즉시 보이게 한다**: 점을 움직이면 식과 값이 바로 바뀐다.
3. **오류를 학습 기회로 만든다**: 단순 오답 표시보다 어디를 다시 봐야 하는지 알려준다.
4. **수업 중 사용이 쉬워야 한다**: 로그인 없이 링크와 코드만으로 시작한다.
5. **모바일보다 태블릿·노트북 우선**: 좌표평면 조작의 정확도를 위해 큰 화면을 우선 고려한다.

## 9. 이번 프로젝트에서 제외하는 기능

- 포물선, 타원, 쌍곡선
- 연립부등식 영역 색칠
- 일반적인 궤적 탐구
- 학생 문제 제작 기능
- AI 자동 피드백 기능

> 위 기능은 매력적이지만 이번 수업 목표를 흐릴 수 있으므로 다루지 않는다. 개발과 수업은 직선과 원의 방정식 이해에 집중한다.
