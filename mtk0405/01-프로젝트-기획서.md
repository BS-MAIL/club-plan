## 1. 기획 배경과 범위

이번 앱은 도형의 방정식 전체를 다루지 않는다. **직선의 방정식과 원의 방정식**에 집중한다. 학생이 한 차시에 너무 많은 개념을 다루지 않고, 다음 두 가지 연결을 확실히 이해하도록 설계한다.

1. **직선**: 기울기와 y절편, 또는 한 점과 기울기가 직선을 어떻게 결정하는가?
2. **원**: 중심과 반지름이 원의 방정식에 어떻게 나타나는가?

실제 수업에서는 학생들이 다음과 같은 어려움을 자주 겪는다.

1. **식과 그래프의 연결 부족**  
   학생들은 `y = ax + b`, `(x-a)^2 + (y-b)^2 = r^2` 같은 식을 외우지만, 계수 변화가 그래프에 어떤 의미를 가지는지 직관적으로 이해하지 못하는 경우가 많다.

2. **계수의 의미 이해 부족**  
   `y = ax + b`에서 `a`, `b`가 무엇을 바꾸는지, `(x-a)^2 + (y-b)^2 = r^2`에서 `a`, `b`, `r`이 무엇을 의미하는지 헷갈린다.

3. **공식 중심 학습의 한계**  
   직선과 원의 방정식은 “그래프의 특징 → 식의 요소 → 다시 그래프 확인”의 순환을 이해해야 하지만, 수업이 공식 적용 문제 풀이로 치우치면 학생의 탐구 경험이 줄어든다.

4. **교사의 학생 사고 과정 파악 어려움**  
   학생이 어떤 값을 조작했는지, 어느 미션에서 오래 머물렀는지, 어떤 개념에서 혼란을 겪었는지 수업 중 실시간으로 파악하기 어렵다.

이에 따라 학생이 직접 조작하며 발견하고, 교사는 학생의 탐구 과정을 데이터로 확인할 수 있는 웹 기반 수학 실험실이 필요하다.

## 2. 프로젝트 목적

### 2.1 교육적 목적

1. 학생이 직선의 방정식을 **기울기와 절편의 관계**로 이해하도록 돕는다.
2. 학생이 원의 방정식을 **중심과 반지름의 관계**로 이해하도록 돕는다.
3. 식을 바꾸면 그래프가 어떻게 변하는지, 그래프를 보면 식의 어떤 부분을 알 수 있는지 양방향으로 연결한다.
4. 직선과 원에 대한 학생의 오개념을 웹앱 조작과 즉각 피드백으로 줄인다.

### 2.2 기술적 목적

1. 브라우저에서 바로 실행 가능한 인터랙티브 수학 웹앱을 만든다.
2. 교사가 미션을 쉽게 추가·수정할 수 있는 구조를 설계한다.
3. 학생 활동 로그를 익명으로 수집하여 수업 개선에 활용한다.
4. 불필요한 확장 기능을 줄이고, 2차시 수업에서 바로 사용할 수 있는 최소 기능에 집중한다.

## 4. 핵심 콘셉트

### 4.1 학생 경험 흐름

```text
직선 또는 원 문제 상황 제시
  ↓
좌표평면에서 직선 또는 원을 직접 조작
  ↓
변화 관찰 및 규칙 추측
  ↓
식 입력 또는 식 완성
  ↓
웹앱이 즉시 그래프와 피드백 제공
  ↓
탐구 기록 작성
  ↓
개념 일반화 및 발표
```

### 4.2 앱의 핵심 가치

| 가치 | 설명 |
|:---|:---|
| 조작 가능성 | 학생이 기울기, 절편, 중심, 반지름을 직접 바꾼다. |
| 즉각 피드백 | 입력한 식과 실제 그래프가 바로 연결된다. |
| 탐구 중심 | 정답을 맞히는 것보다 규칙을 발견하는 과정을 중시한다. |
| 데이터 기반 | 학생의 시도, 체류 시간, 미션 완료 여부를 수업 개선에 활용한다. |
| 집중성 | 직선과 원 외의 개념은 다루지 않아 수업 목표를 분명히 한다. |

## 5. 주요 학습 모듈

### 모듈 A. 직선의 방정식 실험실

- 두 점을 움직이면 직선이 자동 생성
- 기울기, y절편, 한 점과 기울기 형태 비교
- 미션 예시: “점 A를 지나고 기울기가 2인 직선을 만들어라.”
- 식 입력 후 그래프와 일치 여부 확인

### 모듈 B. 원의 방정식 실험실

- 중심과 반지름 슬라이더 조작
- `(x-a)^2 + (y-b)^2 = r^2`에서 `a`, `b`, `r`의 역할 탐구
- 표준형 방정식만 다루고, 일반형 변환은 이번 수업 범위에서 제외

### 모듈 C. 식-그래프 연결 연습

- 그래프를 보고 방정식 맞히기
- 조건을 읽고 그래프 만들기
- 직선과 원의 식에서 틀린 부분 찾기
- 기울기·절편·중심·반지름에 대한 오개념 피드백 받기

## 7. 수업 활용 시나리오

### 시나리오 1. 개념 도입형

학생들이 `y = ax + b`의 `a`, `b` 값을 슬라이더로 바꾸며 직선이 회전하거나 평행이동하는 모습을 관찰한다. 이후 기울기와 y절편의 의미를 말로 정리한다.

### 시나리오 2. 개념 정리형

학생들이 원의 중심을 움직이고 반지름을 바꾸며 `(x-a)^2 + (y-b)^2 = r^2`의 각 요소가 그래프에 어떻게 반영되는지 확인한다. 특히 중심 좌표의 부호와 반지름 제곱을 집중적으로 다룬다.

### 시나리오 3. 정리·확인형

학생들이 그래프를 보고 직선 또는 원의 방정식을 입력하거나, 주어진 식을 보고 그래프를 예측한다. 교사는 오답 유형을 확인해 보충 설명을 제공한다.

## 8. 기대 효과

1. 학생은 직선과 원의 방정식을 암기 대상이 아니라 **그래프의 특징을 표현하는 언어**로 이해한다.
2. 교사는 학생의 탐구 과정을 관찰 가능한 데이터로 확보한다.
3. 수업은 강의 중심에서 실험·발견·공유 중심으로 전환된다.
4. 개발된 앱은 MAIL 동아리의 디지털 조작형 수학 탐구 컴포넌트 사례로 확산 가능하다.
