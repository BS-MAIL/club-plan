# MAIL 동아리 10차시 활동 계획서

> **최종 목표**: 각 동아리원(교사)별 **2차시 수업 지도안 + 실제 수업 수행 결과물**  
> **활동 기간**: 2026년 3월 ~ 2027년 2월 (총 10차시)  
> **대상**: 부산광역시 AI·디지털 교사 동아리 MAIL 회원

---

## 활동 개요

### 동아리 활동의 본질
MAIL 동아리는 "교사가 AI를 도구로 활용하여 자신만의 디지털 수업을 설계하고 실현하는 연구 공동체"입니다.

### 최종 산출물 정의
| 구분 | 내용 | 형식 |
|:---|:---|:---|
| **산출물 A** | 2차시 수업 지도안 (상세안) | 문서 (Markdown/워드) |
| **산출물 B** | 실제 수업 수행 결과 | 영상/사진/학생작품/데이터 등 |
| **산출물 C** | 수업 개선 및 소감문 | 문서 |

> 각 교사는 자신의 **담당 과목/학년/희망 주제**를 자율 선택하여 위 산출물을 완성합니다.

---

## 10차시 활동 로드맵

```
[1~2차시]  기획 단계: 주제 선정 및 목표 설정
[3~4차시]  설계 단계: AI 활용 방향 결정 및 지도안 초안
[5~6차시]  개발 단계: 자료 개발 및 교구 구현
[7~8차시]  수행 단계: 현장 적용 및 데이터 수집
[9~10차시] 분석 단계: 결과 분석 및 공유
```

### 🎯 1차시: 동아리 출범 및 주제 탐색

| 항목 | 내용 |
|:---|:---|
| **단계** | 기획 - 주제 선정 |
| **목표** | 동아리원 간 역량 공유 및 개별 수업 주제 후보 도출 |

#### 활동 내용
1. **아이스브레이킹**: 각자의 학교, 과목, 디지털 교육 경험 소개 (30분)
2. **역량 맵 작성**: 
   - 내가 잘하는 과목/단원
   - AI/디지털 도구 활용 경험
   - 하고 싶은 실험/탐구 주제
3. **주제 후보 브레인스토밍**: 
   - "내 수업에서 학생들이 가장 어려워하는 개념은?"
   - "AI/디지털로 해결하고 싶은 교육적 문제는?"
4. **1차 주제 후보 3개씩 선정**

#### 산출물
- `역량 맵` (개인별)
- `주제 후보 리스트` (3개)

### 🎯 2차시: 주제 확정 및 수업 목표 설정

| 항목 | 내용 |
|:---|:---|
| **단계** | 기획 - 목표 결정 |
| **목표** | 최종 주제 1개 확정 및 2차시 수업의 성취기준·목표 설정 |

#### 활동 내용
1. **주제 피드백**: 동아리원 간 1:1 피드백 (주제의 적절성, 실현 가능성) (30분)
2. **2022 개정 교육과정 성취기준 매핑**:
   - 내 주제와 연결되는 성취기준 찾기
   - 단원/성취기준/평가기준 정리
3. **2차시 수업 목표 설계**:
   - 1차시: 도입·탐구 (학생이 AI/디지털 도구로 탐구하는 활동)
   - 2차시: 정리·확장 (탐구 결과를 개념화하고 적용하는 활동)
4. **예상 학생 수준 및 어려움 예측**

#### 산출물
- `주제 확정서` (성취기준 포함)
- `2차시 수업 목표안`

### 🤖 3차시: AI 활용 방향 결정 I - 도구 탐색

| 항목 | 내용 |
|:---|:---|
| **단계** | 설계 - AI 이용 방향 결정 |
| **목표** | 자신의 수업 목표에 맞는 AI/디지털 도구 탐색 및 선정 |

#### 활동 내용
1. **도구 쇼케이스**: 동아리원 각자 알고 있는 AI 도구 1개씩 소개 (30분)
   - 생성형 AI (ChatGPT, Claude, Gemini 등)
   - 코딩 비서 (OpenCode, GitHub Copilot 등)
   - 시각화/시뮬레이션 도구 (GeoGebra, Desmos, PhET 등)
   - 자동화 도구 (n8n, Zapier 등)
2. **도구 평가 매트릭스**:
   | 평가 항목 | 가중치 | 도구A | 도구B | 도구C |
   |:---|:---:|:---:|:---:|:---:|
   | 교육적 적합성 | 30% | | | |
   | 학생 접근성 | 25% | | | |
   | 데이터 수집 가능성 | 20% | | | |
   | 교사 구현 난이도 | 15% | | | |
   | 확장성/공유 가능성 | 10% | | | |
3. **1차 도구 선정 및 활용 아이디어 스케치**

#### 산출물
- `도구 평가 매트릭스` (개인별)
- `선정 도구 및 활용 아이디어 노트`

### 🤖 4차시: AI 활용 방향 결정 II - 학습 설계

| 항목 | 내용 |
|:---|:---|
| **단계** | 설계 - 지도안 초안 작성 |
| **목표** | AI 도구를 어떻게 학생 활동에 녹일지 구체적 학습 설계 |

#### 활동 내용
1. **학습 설계 패턴 공유**:
   - **패턴 A**: AI가 "탐구 도구" (학생이 AI로 가설 검증)
   - **패턴 B**: AI가 "창작 도구" (학생이 AI로 콘텐츠 제작)
   - **패턴 C**: AI가 "피드백 도구" (AI가 학생 답안 분석 및 환류)
   - **패턴 D**: AI가 "데이터 수집 도구" (학습 과정 데이터 자동 수집)
2. **자신의 수업에 패턴 적용**: 
   - 1차시에 어떤 패턴을 어디에 넣을지
   - 2차시에 어떤 패턴을 어디에 넣을지
3. **지도안 초안(1차시) 작성**: 
   - 도입(5분) - 전개(25분) - 정리(10분) 구조
   - 각 단계에서 AI 도구의 역할 명시
4. **동아리원 간 피드백**

#### 산출물
- `학습 설계 패턴 적용안`
- `1차시 지도안 초안`

### 🛠️ 5차시: 자료 개발 I - 인터랙티브 자료 제작

| 항목 | 내용 |
|:---|:---|
| **단계** | 개발 - 지도안 및 자료 개발 |
| **목표** | 1차시 수업용 인터랙티브 자료/교구 개발 실습 |

#### 활동 내용
1. **개발 역량 점검**: 
   - 코딩 가능 여부, 도구 숙련도 확인
   - 팀 내 협업 파트너 결정 (개발+콘텐츠 조합)
2. **자료 개발 실습** (90분 집중 작업):
   - **옵션 1**: OpenCode/AI 코딩 비서로 웹 인터랙티브 교구 개발
   - **옵션 2**: 기존 플랫폼(GeoGebra, Desmos, Canva 등) 활용
   - **옵션 3**: 프롬프트 엔지니어링으로 AI 활동 시나리오 설계
3. **개발 중간 점검**: 동아리원 간 1:1 테스트 및 피드백

#### 산출물
- `1차시 인터랙티브 자료` (웹/파일/프롬프트 등)
- `개발 과정 기록` (어려움, 해결 방법)

### 🛠️ 6차시: 자료 개발 II - 2차시 자료 및 평가 도구

| 항목 | 내용 |
|:---|:---|
| **단계** | 개발 - 지도안 및 자료 개발 |
| **목표** | 2차시 자료 완성 및 학습 데이터 수집/평가 도구 준비 |

#### 활동 내용
1. **2차시 지도안 작성**: 
   - 1차시 탐구 결과를 바탕으로 한 개념 정리 활동
   - 학생 간 공유/발표 구조 설계
2. **평가 도구 개발**:
   - **과정 중심 평가**: 탐구 과정 체크리스트, 협업 루브릭
   - **결과 중심 평가**: AI 생성물 평가 기준, 개념 적용 문제
   - **데이터 수집**: 학생 반응 설문(체감 난이도, 흥미도), 학습 행동 로그
3. **2차시 자료 완성 및 통합 테스트**

#### 산출물
- `2차시 지도안 완성본`
- `평가 도구` (루브릭, 설문지, 데이터 수집 시트)
- `통합 2차시 수업 자료 패키지`

|:---|:---|

2. **분석 프레임워크 적용**:
   | 분석 영역 | 질문 예시 |

3. **개선안 작성**: 
   - "다음에 이 수업을 한다면 바꿀 3가지"
   - "다른 교사에게 전하고 싶은 팁"

#### 산출물
- `수업 데이터 분석 보고서`

## 개인별 산출물 체크리스트

각 동아리원은 10차시 활동을 통해 아래 산출물을 완성합니다.

### 필수 산출물
- [ ] **2차시 수업 지도안** (상세안)
  - [ ] 성취기준 및 평가기준 명시
  - [ ] AI/디지털 도구 활용 계획
  - [ ] 학생 활동 시나리오
  - [ ] 평가 도구 (루브릭/체크리스트)
- [ ] **수업 수행 증거 자료**
  - [ ] 수업 영상 또는 사진 (학생 얼굴 가림 처리)
  - [ ] 학생 탐구/창작 결과물
  - [ ] 학생 설문 결과
- [ ] **소감문 및 개선안**
  - [ ] 수업 설계 소감
  - [ ] 현장 적용 소감
  - [ ] 개선 방향 및 재설계 아이디어

### 선택 산출물 (희망자)
- [ ] 웹 인터랙티브 교구 (OpenCode 등으로 개발)
- [ ] AI 프롬프트 가이드
- [ ] 학습 데이터 분석 보고서
- [ ] 동영상 수업 사례 (5분 내외)

## 활동 운영 규칙

### 1. 자율 선택 원칙
- 각 교사는 **자신의 과목, 학년, 관심 주제**를 자율 선택
- AI 활용 방식도 자율 (생성형 AI, 코딩, 데이터 분석 등)

### 2. 협업 원칙
- 개발이 어려운 교사는 개발팀 교사와 1:1 매칭
- 콘텐츠 설계가 어려운 교사는 콘텐츠팀 교사와 협업

### 3. 데이터 윤리
- 학생 개인정보는 반드시 익명화/가명화
- 수업 영상/사진 사용은 학생/학부모 동의 확보
- AI 생성 데이터는 학교 내부용으로만 활용

### 4. 오픈 소스 정신
- 개발한 자료는 동아리 내에서 공유
- 부산교수학습샘터 보급을 위한 정제 버전 제작

## 예시: 김규봉 교사의 2차시 수업 산출물

| 항목 | 내용 |
|:---|:---|
| **과목** | 수학 (공통수학1) |
| **단원** | 다항식의 곱셈 |
| **성취기준** | 다항식의 곱셈을 이해하고 계산할 수 있다 |
| **1차시** | OpenCode 기반 다항식 곱셈 시각화 탐구 |
| **2차시** | 탐구 결과를 바탕으로 한 조립제법 적용 및 문제 해결 |
| **AI 활용** | OpenCode로 인터랙티브 다항식 곱셈 컴포넌트 개발, 학생이 변수 조작하며 원리 발견 |
| **데이터 수집** | 문항별 체류 시간, 체감 난이도, 탭 전환 횟수 |
