## 1. 수업 설계 방향

본 수업은 특정 소재 하나를 해결하는 프로젝트가 아니라, 공통수학2 전체를 관통하는 Big Idea에서 출발하여 학생들이 학교와 지역사회의 문제를 직접 발견하고 수학적으로 모델링하는 장기 CBL 수업이다.

학생들은 공간, 조건, 관계, 변화와 관련된 문제를 선택하고, 공통수학2의 도형의 방정식, 집합과 명제, 함수와 그래프 개념을 활용하여 문제를 분석한다. AI는 정답 생성 도구가 아니라 문제 발견, 질문 정교화, 모델 검토, 반례 탐색, 발표 피드백을 돕는 보조 도구로 사용한다.

### CBL 성격을 분명히 하는 장치

- 교사가 처음부터 하나의 문제를 주지 않고 Big Idea에서 학생들이 문제 후보를 발견한다.
- 여러 Challenge가 나올 수 있도록 허용한다.
- 수학 개념은 문제 해결 도구로 선택·활용된다.
- AI는 답 생성기가 아니라 질문·검증·성찰 도구로 사용된다.
- 최종 산출물이 보고서 제출에서 끝나지 않고 학교 공동체에 공유된다.
- 모델의 한계와 현실 적용 가능성을 반드시 성찰한다.

### 4.2 집합과 명제

| 개념 | 프로젝트 연결 |
|---|---|
| 집합 | 연결 가능 구역, 안전 구역, 접근 가능 구역, 위험 구역 표현 |
| 합집합 | 여러 시설 또는 여러 공유기가 커버하는 전체 영역 표현 |

| 여집합 | 사각지대, 미충족 구역 표현 |
| 명제 | 이 조건을 만족하면 안전하다, 연결된다, 접근 가능하다는 판단 기준 표현 |
| 조건문 | 수학 모델 안에서 참인 기준과 현실에서의 예외 비교 |
| 역·대우 | 문제 판단 기준의 타당성 검토 |

예시:

```text
A = 와이파이 연결 가능 구역
B = 학생들이 자주 사용하는 공간
A ∩ B = 연결이 필요한 공간 중 실제 연결되는 구역
B - A = 디지털 학습 사각지대
```

예시 명제:

```text
p: 점 P가 안전 반경 안에 있다.
q: 점 P는 실제로 안전하다.

수학 모델에서는 p → q라고 가정할 수 있지만,
현실에서는 장애물, 혼잡, 시야, 이동 시간 때문에 항상 참은 아닐 수 있다.
```

## 6. 단계별 수업 스케치

### 6.1 1~2주: Engage - Big Idea 탐색

| 항목 | 내용 |
|---|---|
| 핵심 질문 | 학교에서 잘 보이지 않지만 생활에 영향을 주는 문제는 무엇인가? |
| 활동 | 학교 공간 관찰, 불편 경험 공유, AI로 문제 후보 확장 |
| 수학 연결 | 좌표, 거리, 조건, 관계, 변화 |
| AI 활용 | 문제 후보 생성, 탐구 질문 다듬기, 사례 조사 |
| 산출물 | 개인 문제 발견 카드, 모둠별 관심 주제 목록 |

교사 발문:

- 보이지 않지만 우리 생활을 바꾸는 것은 무엇인가?
- 연결, 안전, 접근성, 이동, 환경은 어떻게 측정할 수 있을까?
- 문제를 수학적으로 다루려면 무엇을 좌표, 거리, 조건, 함수로 바꾸어야 할까?

### 6.2 3~5주: Investigate 1 - 공간을 좌표와 방정식으로 보기

| 항목 | 내용 |
|---|---|

| 활동 | 학교 약식 지도 좌표화, 주요 시설 좌표 표시, 거리와 범위 분석 |
| 수학 연결 | 내분점, 직선의 방정식, 점과 직선 사이 거리, 원의 방정식 |
| AI 활용 | 좌표화 기준 점검, 분석 가능 질문 생성, 설명 피드백 |
| 산출물 | 좌표화 지도, 1차 공간 분석표 |

가능한 미니 과제:

- 두 공간 사이의 균형 위치 찾기
- 주요 복도를 직선으로 나타내기
- 시설 접근 가능 범위를 원으로 나타내기
- 사각지대 후보 찾기

### 6.3 6~8주: Investigate 2 - 조건과 판단 기준 만들기

| 항목 | 내용 |
|---|---|
| 핵심 질문 | 어떤 조건을 만족해야 연결된다, 안전하다, 접근 가능하다고 말할 수 있을까? |
| 활동 | 판단 기준 만들기, 집합으로 구역 분류, 명제의 참거짓 검토 |
| 수학 연결 | 집합, 합집합, 교집합, 여집합, 명제, 조건문 |
| AI 활용 | 기준의 타당성 검토, 반례 찾기, 명제 문장 다듬기 |
| 산출물 | 조건 기준표, 집합 표현 지도, 명제 검증 기록 |

예시 활동:

```text
A = 특정 시설에서 5분 이내 접근 가능한 구역
B = 학생들이 자주 사용하는 구역
B - A = 접근성 개선이 필요한 구역
```

학생들은 모델 안에서 참인 명제가 현실에서는 예외를 가질 수 있음을 분석한다.

### 6.4 9~12주: Investigate 3 - 관계와 변화를 함수로 보기

| 항목 | 내용 |
|---|---|
| 핵심 질문 | 거리, 시간, 신호, 소음, 이용량은 어떤 관계로 변화하는가? |
| 활동 | 함수 모델 만들기, 표와 그래프 작성, 증가·감소 경향 분석 |
| 수학 연결 | 함수, 합성함수, 역함수, 유리함수, 무리함수 |
| AI 활용 | 함수 모델 후보 비교, 그래프 해석 피드백, 모델 한계 조사 |
| 산출물 | 함수 모델링 보고서, 그래프, 모델 비교표 |

핵심 연결:

```text
현실의 양 → 함수 관계 → 그래프 해석 → 기준 설정 → 개선안 판단
```

### 6.5 13~15주: Act - 개선안 설계와 AI 검증

| 항목 | 내용 |

| 수학 연결 | 도형의 이동, 집합, 함수, 그래프 통합 |
| AI 활용 | 개선안 검토, 예상 반론 생성, 발표 피드백 |
| 산출물 | 최종 개선안, 시각화 자료, AI 활용 및 검증 기록 |

개선안 예시:

- 와이파이 공유기 또는 중계기 위치 변경
- 소화기, AED, 안내 표지 추가 위치 제안
- 휴게공간 또는 학습공간 접근성 개선
- 조도·소음·혼잡 문제 개선
- 이동 약자를 고려한 동선 제안

### 6.6 16~17주: Share & Reflect - 발표와 성찰

## 10. 성취수준 도달 설명

본 수업은 공통수학2의 개별 성취기준을 분절적으로 다루지 않고, 학교와 지역사회의 문제를 수학적으로 모델링하는 장기 CBL 프로젝트로 구성한다. 학생들은 공간 문제를 좌표와 도형의 방정식으로 표현하고, 조건과 판단 기준을 집합과 명제로 정리하며, 변화와 관계를 함수와 그래프로 해석한다.
