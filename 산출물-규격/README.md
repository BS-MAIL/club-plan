# MAIL 동아리 산출물 규격 가이드

> **목적**: 각 동아리원(교사)이 10차시 활동을 통해 완성할 개인별 산출물의 표준 양식을 정의합니다.
> **대상**: MAIL(Math-AI Interactive Lab) 동아리 전체 회원

---

## 산출물 개요

각 교사는 아래 3가지 핵심 산출물을 반드시 완성해야 합니다.

| 번호 | 산출물명 | 형식 | 용량/분량 | 템플릿 파일 |
|:---:|:---|:---:|:---|:---|
| **A** | 2차시 수업 지도안 (상세안) | Markdown / Word | 5~10페이지 | `01-2차시-수업지도안-템플릿.md` |
| **B** | 수업 수행 증거 자료 | 폴터 (영상/사진/문서) | 500MB 이내 | `02-수업수행-증거자료-수집양식.md` |
| **C** | 소감문 및 개선안 | Markdown / Word | 2~5페이지 | `03-소감문-및-개선안-템플릿.md` |

---

## 산출물 A: 2차시 수업 지도안

### 필수 포함 항목
- [ ] 수업 개요 (과목, 학년, 단원, 성취기준)
- [ ] 수업 설계 의도 (학습자 분석, 교육적 문제, AI 활용 근거)
- [ ] 1차시 지도안 (도입-전개-정리 구조, AI 도구 활용 명시)
- [ ] 2차시 지도안 (도입-전개-정리 구조, 평가 포함)
- [ ] 평가 계획 (과정/결과 중심 평가, 루브릭)
- [ ] 수업 자료 목록 (개발한 자료, AI 프롬프트 등)
- [ ] 예상 질문 및 대응 전략

### 작성 팁
- AI 활용 부분은 **구체적으로** 작성 ("ChatGPT를 사용한다"가 아닌 "학생이 ChatGPT에게 ~라고 프롬프트를 입력하여 ~를 탐구한다")
- 학생 활동은 **행동 중심**으로 서술 ("듣는다"보다 "탐구한다", "발견한다", "적용한다")

---

## 산출물 B: 수업 수행 증거 자료

### 구성 요소
1. **수업 영상/사진** (필수)
   - 학생 얼굴은 반드시 모자이크/가림 처리
   - 각 차시별 최소 3장 이상의 사진 또는 5분 이상의 영상
   - 파일명: `YYYY-MM-DD_차시_단계_번호`

2. **학생 활동 결과물** (필수)
   - AI/디지털 도구 생성물 (텍스트, 이미지, 코드 등)
   - 탐구/학습 기록물
   - 개인정보 제거 후 수집

3. **학생 설문 결과** (필수)
   - 최소 4개 문항 (이핵도, 편의성, 흥미도, 도움 정도)
   - 응답률 70% 이상 권장
   - 익명성 보장

4. **학습 데이터 로그** (선택)
   - 자동 수집 가능한 경우 (체류 시간, 이탈 행동 등)

### 보관 규칙
```
/산출물-B_수행증거자료
  ├── /01_수업영상및사진
  │     ├── YYYY-MM-DD_1차시_도입_01.jpg
  │     └── ...
  ├── /02_학생작품및결과물
  │     └── ...
  ├── /03_학생설문결과
  │     └── 설문결과_통계.xlsx
  ├── /04_학습데이터로그 (해당 시)
  │     └── ...
  └── /05_교사관찰기록
        └── 수업관찰기록.md
```

## 산출물 C: 소감문 및 개선안

### 필수 포함 항목
- [ ] 수업 설계 소감 (주제 선택 계기, 도구 선택 이유, 어려움과 극복)
- [ ] 현장 적용 소감 (기대 vs 실제, 긍정적/아쉬운 순간, 학생 반응)
- [ ] AI 도구 활용 평가 (적합성 5점 척도 평가, 장점/한계)
- [ ] 개선 방향 ("다음에 바꿀 3가지", 재설계 아이디어, 동료 교사용 팁)
- [ ] 동아리 활동 전반 소감 (평가, 제안, 한마디)

### 작성 팁
- **솔직하게** 작성 (잘된 점만 쓰지 말고, 실패와 시행착오도 기록)
- **구체적으로** 작성 ("학생들이 좋아했다"보다 "학생 3명이 ~라고 말했다")
- **성장 중심**으로 작성 ("잘했다/못했다"보다 "배웠다/발전시키고 싶다")

## 파일 제출 규칙

### 개인별 폴터 구조
```
/MAIL_산출물_[이름]_[학교]
  ├── /01_수업지도안
  │     └── 2차시-수업지도안-[주제명].md
  ├── /02_수행증거자료
  │     ├── /01_수업영상및사진
  │     ├── /02_학생작품및결과물
  │     ├── /03_학생설문결과
  │     └── /04_교사관찰기록
  ├── /03_소감문및개선안
  │     └── 소감문-[이름].md
  └── README.md (간단한 인덱스)
```

### 제출 시점
| 산출물 | 임시 제출 | 최종 제출 |
|:---|:---|:---|
| A. 수업지도안 | 6차시 후 (초안) | 8차시 후 (수정안) |
| B. 수행증거자료 | 7차시 후 (1차시 자료) | 8차시 후 (전체) |
| C. 소감문 | - | 10차시 후 |

### 제출 방식
- GitHub 리포지토리 또는 공유 클라우드 폴더
- 파일명은 영문/숫자/언더스코어 사용 (예: `2026-05-20_1차시_도입_01.jpg`)

## 데이터 윤리 및 개인정보 보호

### 반드시 지켜야 할 사항
1. **학생 얼굴**: 모든 사진/영상에서 학생 얼굴은 모자이크 또는 가림 처리
2. **개인정보**: 학생 이름, 학번, 학교명 등은 가명화 또는 제거
3. **동의**: 수업 영상/사진 촬영은 학생/학부모 동의를 사전에 확보
4. **AI 데이터**: 학생이 AI 도구에 입력한 데이터는 외부 유출 금지
5. **보관**: 수업 종료 후 불필요한 데이터는 즉시 폐기

### 동의서 양식 (별첨)
- `부록-수업촬영동의서-가정통신문.md`

| **데이터 기반** | 다차원 데이터를 체계적으로 수집·분석 | 주요 데이터를 적절히 수집·분석 | 기본적인 데이터 수집 | 데이터 수집이 미흡 | 데이터 수집이 거의 없음 |
