## 1. 동아리 소개

### 1.1 동아리명
**MAIL (Math-AI Interactive Lab)**

### 1.2 이름의 의미
- **M**ath: 수학교육의 본질 - 탐구와 발견
- **AI**: 인공지능 기술의 교육적 활용
- **I**nteractive: 학생-교구-데이터의 쌍방향 상호작용
- **Lab**: 실험하고 검증하는 연구 공동체

### 1.3 비전
> "AI는 도구이며, 교사가 설계한 진짜 탐구의 주인은 학생이다"

MAIL은 AI와 디지털 도구를 활용하되, 수업의 중심을 기술이 아니라 **학생의 질문, 조작, 탐구, 성찰**에 둔다. 2026학년도 활동은 CBL(Challenge-Based Learning, 도전 중심 학습) 관점을 반영하여, 각 교사가 실제 수업에서 실행 가능한 2차시 탐구 수업을 설계·수행·분석하는 것을 목표로 한다.

## 2. 연구 주제

### 2.1 핵심 연구 영역

| 번호 | 연구 주제 | 상세 내용 |
|:---:|:---|:---|
| 가 | **CBL 기반 디지털 수학 탐구 설계** | Big Idea, Essential Question, Challenge 구조를 활용하여 학생이 실제 문제를 수학적으로 탐구하는 2차시 수업 설계 |
| 나 | **AI 협업형 인터랙티브 교구 개발** | OpenCode와 생성형 AI를 활용하여 교사가 수업 의도에 맞는 웹 교구, 활동지, 프롬프트 세트를 제작 |
| 다 | **학습 과정 데이터 수집·분석** | 활동지, AI 사용 기록, 설문, 학습 로그를 바탕으로 학생의 탐구 과정과 이해 변화를 분석 |
| 라 | **AI 윤리·출처 검증 기반 과정 평가** | AI 사용 목적, 프롬프트, 결과 수정, 출처 확인, 개인정보 보호를 포함한 과정 중심 평가 모델 개발 |

### 2.2 CBL-AI 운영 프레임

각 수업은 다음 흐름으로 설계한다.

| CBL 단계 | MAIL 적용 방식 | 주요 산출물 |
|---|---|---|
| Big Idea | 수학 개념과 실제 맥락을 연결하는 큰 주제 선정 | 주제 설명, 성취기준 매핑 |
| Essential Question | 학생이 탐구할 핵심 질문 작성 | 탐구 질문, 수업 목표 |
| Challenge | 2차시 안에서 해결 가능한 도전 과제 제시 | Challenge 문장, 학생 과제 |
| Investigation | AI와 디지털 도구를 활용한 조사·조작·분석 | 활동지, AI 사용 기록, 자료 출처 |
| Solution/Action | 학생 산출물 제작, 발표, 설명, 적용 | 학생 작품, 발표 자료 |
| Reflection/Sharing | 성찰, 피드백, 개선안, 공유 자료 정리 | 소감문, 개선안, 보급 자료 |

## 3. 필요성 및 목적

### 3.1 필요성

#### 가. 수학적 주도권 회복
현재의 디지털 수학 교육은 **수동적인 영상 시청**이나 **단순 문항 풀이**에 머무르는 경우가 많다. 학생들이 스스로 변수를 조작하고 원리를 발견하는 **진짜 탐구 환경**이 필요하다.

#### 나. CBL 기반 실제 문제 해결 경험
CBL은 실제 세계의 문제, 협업, 기술 활용, 창의적 해결을 강조하는 학습 설계 방식이다. MAIL은 이를 수학 수업에 적용하여 학생이 단순히 개념을 듣는 것이 아니라, 문제를 정의하고 해결 과정을 설명하는 경험을 제공하고자 한다.

#### 다. 교사의 에듀테크 주권 확보
외부 유료 솔루션에 의존하기보다, 교사가 자신의 수업 의도에 맞는 인터랙티브 교구와 활동 자료를 직접 제작·수정·배포할 수 있어야 한다. AI 코딩 비서와 생성형 AI는 이 과정을 가속하는 협업 도구가 될 수 있다.

#### 라. 평가의 정교화
학생의 탐구 과정, AI 사용 방식, 활동지 기록, 체감 난이도, 발표 내용 등을 데이터화하면 결과 중심 평가를 넘어 과정 중심 환류가 가능하다. 특히 생성형 AI 시대에는 최종 답안보다 **문제 정의, 출처 검증, 수정 과정, 성찰**을 평가하는 체계가 필요하다.

### 3.2 목적

1. **학생 중심 탐구 환경 조성**: 디지털 샌드박스와 AI 도구를 활용해 학생이 주도적으로 수학적 원리를 발견하도록 지원
2. **CBL 기반 수업 설계 역량 강화**: Big Idea, Essential Question, Challenge 구조를 활용한 2차시 수업 설계 전문성 개발
3. **교사 디지털 제작 역량 강화**: OpenCode와 생성형 AI를 활용한 맞춤형 교구·활동지·평가 도구 제작
4. **데이터 기반 교수학습 개선**: 학습 과정 데이터를 활용한 과정 중심 평가와 맞춤형 환류 모델 구축
5. **오픈 소스 기반 확산**: 개발 자료를 동아리 내외로 공유하여 지속 가능한 디지털 교육 혁신 문화 조성

## 5. 기대 효과

### 5.1 교사 주도적 디지털 혁신
AI에 의존하는 수업이 아니라, 교사가 설계한 CBL 구조와 인터랙티브 교구를 통해 수업의 주도권을 확보한다.

### 5.2 수학적 사고의 시각화
학생들이 수학적 개념을 동적으로 조작하고, 자신의 탐구 과정을 설명하며, 개념을 실제 맥락과 연결한다.

### 5.3 AI 활용 역량의 실제화
AI를 단순 검색·대필 도구로 쓰는 것이 아니라, 질문 생성, 교구 개발, 피드백, 데이터 분석을 돕는 협업 도구로 활용한다.

### 5.4 데이터 기반 과정 중심 평가
학생의 탐구 궤적, AI 사용 기록, 설문, 발표, 성찰을 함께 분석하여 과정 중심 평가와 맞춤형 환류 모델을 제시한다.

### 5.5 오픈 소스 기반 확산
특정 유료 솔루션에 종속되지 않는 자료와 교구를 개발하고, 부산교수학습샘터 및 디지털 교육 현장지원단 네트워크를 통해 공유한다.

## 6. 활동 원칙

1. **학생 주도성**: 학생이 문제를 정의하고 탐구하고 설명하는 과정을 수업의 중심에 둔다.
2. **AI 도구성**: AI는 사고를 대체하지 않고 질문, 제작, 피드백, 분석을 돕는 도구로 사용한다.
3. **출처 검증**: AI가 제시한 정보는 원문 자료, 공식 문서, 논문 등으로 확인한다.
4. **개인정보 보호**: 학생 개인정보와 민감 정보는 AI 도구에 입력하지 않고, 공유 자료는 익명화한다.
5. **과정 중심 평가**: 최종 산출물보다 문제 정의, 탐구, 실패, 수정, 성찰의 과정을 중시한다.
6. **오픈 소스 정신**: 개발된 교구와 가이드는 가능한 범위에서 공개하여 공유 문화를 조성한다.
7. **현장 중심**: 모든 개발은 실제 수업 현장의 필요에서 출발하고, 현장에서 검증한다.
8. **지속 가능성**: 1년 활동으로 끝나지 않고, 다음 해에도 확장 가능한 운영 모델을 구축한다.
