# MAIL 월별 활동 계획

> **활동 기간**: 2026년 3월 ~ 2027년 2월  
> **운영 방식**: 월별 목표 및 과제 중심 자율 진행  
> **대상**: MAIL(Math-AI Interactive Lab) 동아리 전체 회원  
> **최종 목표**: 각 교사별 **CBL 기반 2차시 수업 지도안 + 실제 수행 결과물 + 성찰·개선안**

---

## 활동 개요

### 운영 원칙
- **CBL 기반 설계**: Big Idea → Essential Question → Challenge → Investigation → Solution/Action → Reflection/Sharing 흐름을 수업 설계에 반영합니다.
- **AI 협업 전제**: 자료 조사, 아이디어 생성, 교구 개발, 오류 수정, 피드백 정리, 데이터 분석에 AI를 적극 활용합니다.
- **교사 주도성 유지**: AI는 설계와 제작을 돕는 도구이며, 수업 의도와 최종 판단은 교사가 책임집니다.
- **과정 기록 중시**: 결과물뿐 아니라 AI 사용 기록, 출처 검증, 학생 탐구 과정, 수정 이력을 산출물에 포함합니다.
- **월별 점검**: 매월 초 공통 목표를 공유하고, 매월 말 중간 산출물을 누적합니다.

### 최종 산출물
| 구분 | 내용 | 형식 |
|:---|:---|:---|
| **산출물 A** | CBL 기반 2차시 수업 지도안 | Markdown/워드 |
| **산출물 B** | 실제 수업 수행 결과 | 영상/사진/학생작품/데이터 |
| **산출물 C** | 소감문 및 개선안 | Markdown/워드 |
| **산출물 D** | AI 활용 및 출처 검증 기록 | Markdown/스프레드시트 |

---

## 6월 이후 운영 방향

이미 6월에 진입했으므로 3~5월의 탐색 결과를 바탕으로 **6월부터 실행형 CBL 설계**로 전환합니다. 다만 AI와 함께 작업하므로 시간을 과도하게 압축하지 않고, 다음 원칙으로 운영합니다.

1. **6월은 확정과 초안의 달**: 주제, CBL 구조, AI 활용 장면, 2차시 흐름을 확정합니다.
2. **7월은 개발과 동료 검토의 달**: AI와 함께 자료를 빠르게 만들고, 동아리원 피드백으로 수정합니다.
3. **8월은 검증과 안정화의 달**: 새 기능을 늘리기보다 수업 가능성, 시간 배분, 개인정보·출처 기준을 점검합니다.
4. **9~10월은 실행과 증거 수집의 달**: 2차시 수업을 수행하고 학생 탐구 과정과 데이터를 수집합니다.
5. **11~12월은 분석과 공유의 달**: 수업 데이터를 해석하고 개선안을 작성하여 보급 가능한 자료로 정리합니다.

## CBL-AI 설계 산출물 표준

각 교사는 6월 말까지 아래 항목을 1쪽으로 정리합니다.

```md
1. Big Idea:
2. Essential Question:
3. Challenge:
4. 관련 성취기준:
5. AI/디지털 도구:
6. AI 사용 목적:
7. 수집할 학습 증거:
8. 개인정보·출처 검증 기준:
```

AI 사용 목적은 다음 중 하나 이상으로 명확히 적습니다.

| AI 활용 목적 | 수업 장면 예시 |
|---|---|
| 질문 생성 | 탐구 질문, 반례 질문, 후속 질문 만들기 |
| 자료 조사·요약 | 배경 자료, 사례, 공식 문서 요약 |
| 교구 개발 | 웹 교구 코드 초안, 오류 수정, UI 문구 정리 |
| 피드백 | 학생 설명문, 발표문, 활동지 답안 개선 |
| 데이터 분석 | 설문, 활동 로그, 체감 난이도 응답 정리 |
| 평가 보조 | 루브릭 초안, 자기평가 문항, 성찰 질문 생성 |

## 운영 시 유의사항

- 학생 개인정보와 민감 정보는 AI 도구에 입력하지 않습니다.
