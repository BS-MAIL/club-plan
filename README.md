# MAIL (Math-AI Interactive Lab) 동아리 활동 계획

> **부산광역시교육청 AI·디지털 교사 동아리**  
> **신청일**: 2026년 3월 27일  
> **대표**: 김규봉 (부산소프트웨어마이스터고등학교)

---

## 📌 동아리 소개

**MAIL(Math-AI Interactive Lab)** 은 부산광역시교육청 디지털미래교육과 소속 AI·디지털 교사 연구 동아리입니다.

### 이름의 의미
- **M**ath: 수학교육의 본질 - 탐구와 발견
- **AI**: 인공지능 기술의 교육적 활용
- **I**nteractive: 학생-교구-데이터의 쌍방향 상호작용
- **Lab**: 실험하고 검증하는 연구 공동체

### 비전
> "AI는 도구이며, 교사가 설계한 진짜 탐구의 주인은 학생이다"

MAIL은 AI와 디지털 도구를 활용하되, 수업의 중심을 기술이 아니라 **학생의 질문, 조작, 탐구, 성찰**에 둡니다. 2026학년도 활동은 CBL(Challenge-Based Learning, 도전 중심 학습) 관점을 반영하여, 각 교사가 실제 수업에서 실행 가능한 2차시 탐구 수업을 설계·수행·분석하는 것을 목표로 합니다.

---

## 🎯 최종 목표

각 동아리원(교사)은 활동 기간 동안 아래 **4가지 핵심 산출물**을 완성합니다.

| 산출물 | 내용 | 형식 |
|:---:|:---|:---:|
| **A** | CBL 기반 2차시 수업 지도안 | Markdown / Word |
| **B** | 실제 수업 수행 결과 | 영상 / 사진 / 학생 작품 / 데이터 |
| **C** | 소감문 및 개선안 | Markdown / Word |
| **D** | AI 활용 및 출처 검증 기록 | Markdown / 스프레드시트 |

> 각 교사는 자신의 **담당 과목·학년·희망 주제**를 자율 선택하고, Big Idea → Essential Question → Challenge → Investigation → Solution/Action → Reflection/Sharing 흐름에 따라 2차시 수업을 설계·수행·분석합니다.

## 📅 활동 기간

**2026년 3월 ~ 2027년 2월** (12개월)

### 6월 이후 운영 방향

이미 6월에 진입했으므로 3~5월의 탐색 결과를 바탕으로 **6월부터 실행형 CBL 설계**로 전환합니다.

1. **6월은 확정과 초안의 달**: 주제, CBL 구조, AI 활용 장면, 2차시 흐름을 확정합니다.
2. **7월은 개발과 동료 검토의 달**: AI와 함께 자료를 빠르게 만들고, 동아리원 피드백으로 수정합니다.
3. **8월은 검증과 안정화의 달**: 수업 가능성, 시간 배분, 개인정보·출처 기준을 점검합니다.
4. **9~10월은 실행과 증거 수집의 달**: 2차시 수업을 수행하고 학생 탐구 과정과 데이터를 수집합니다.
5. **11~12월은 분석과 공유의 달**: 수업 데이터를 해석하고 개선안을 작성하여 보급 가능한 자료로 정리합니다.

### 월별 흐름

```
[3월]   동아리 출범 및 주제 탐색
[4월]   수업 목표 설정 및 AI 도구 탐색
[5월]   지도안 초안 작성 및 AI 활용 방향 검토
[6월]   CBL 구조 확정 및 2차시 수업 초안 정리
[7월]   AI 협업 기반 수업 자료·교구·평가 도구 개발
[8월]   통합 테스트, 개인정보·출처 기준 점검, 수업 안정화
[9월]   1차시 현장 수업 수행 및 탐구 과정 기록
[10월]  2차시 현장 수업 수행, 발표·공유, 데이터 수집
[11월]  결과 분석 및 개선안 작성
[12월]  성과 정리 및 공유 자료 제작
[1월]   확산 및 보급 활동
[2월]   성과 발표 및 차년도 계획 수립
```

## 📋 연구 주제

MAIL 동아리의 핵심 연구 영역입니다. 각 교사는 이 중 하나 이상을 자신의 수업에 적용합니다.

| 번호 | 연구 주제 | 상세 내용 |
|:---:|:---|:---|
| 가 | **CBL 기반 디지털 수학 탐구 설계** | Big Idea, Essential Question, Challenge 구조를 활용하여 학생이 실제 문제를 수학적으로 탐구하는 2차시 수업 설계 |
| 나 | **AI 협업형 인터랙티브 교구 개발** | OpenCode와 생성형 AI를 활용하여 교사가 수업 의도에 맞는 웹 교구, 활동지, 프롬프트 세트 제작 |
| 다 | **학습 과정 데이터 수집·분석** | 활동지, AI 사용 기록, 설문, 학습 로그를 바탕으로 학생의 탐구 과정과 이해 변화 분석 |
| 라 | **AI 윤리·출처 검증 기반 과정 평가** | AI 사용 목적, 프롬프트, 결과 수정, 출처 확인, 개인정보 보호를 포함한 과정 중심 평가 모델 개발 |

> **참고**: 수학 외 과목 교사도 AI·디지털 도구를 활용한 탐구 중심 수업 설계가 가능합니다.

## 🛠️ 활동 원칙

1. **학생 주도성**: 학생이 문제를 정의하고 탐구하고 설명하는 과정을 수업의 중심에 둡니다.
2. **AI 도구성**: AI는 사고를 대체하지 않고 질문, 제작, 피드백, 분석을 돕는 도구로 사용합니다.
3. **출처 검증**: AI가 제시한 정보는 원문 자료, 공식 문서, 논문 등으로 확인합니다.
4. **개인정보 보호**: 학생 개인정보와 민감 정보는 AI 도구에 입력하지 않고, 공유 자료는 익명화합니다.
5. **과정 중심 평가**: 최종 산출물보다 문제 정의, 탐구, 실패, 수정, 성찰의 과정을 중시합니다.
6. **오픈 소스 정신**: 개발된 교구와 가이드는 가능한 범위에서 공개하여 공유 문화를 조성합니다.
7. **현장 중심**: 모든 개발은 실제 수업 현장의 필요에서 출발하고, 현장에서 검증합니다.
8. **지속 가능성**: 1년 활동으로 끝나지 않고, 다음 해에도 확장 가능한 운영 모델을 구축합니다.

## 📊 기대 효과

- **교사 주도적 디지털 혁신**: AI에 의존하는 수업이 아니라, 교사가 설계한 CBL 구조와 인터랙티브 교구로 수업 주도권 확보
- **수학적 사고의 시각화**: 학생들이 수학적 개념을 동적으로 조작하고 자신의 탐구 과정을 설명하는 수업 구현
- **AI 활용 역량의 실제화**: AI를 단순 검색·대필 도구가 아니라 질문 생성, 교구 개발, 피드백, 데이터 분석을 돕는 협업 도구로 활용
- **데이터 기반 과정 중심 평가**: 학생의 탐구 궤적, AI 사용 기록, 설문, 발표, 성찰을 함께 분석하여 맞춤형 환류 모델 제시
- **오픈 소스 기반 확산**: 특정 유료 솔루션에 종속되지 않는 자료와 교구를 개발하고 교육 현장에 공유
