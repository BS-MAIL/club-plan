# 대푯값 CBL 2차시 수업지도안

## 1. 수업 개요

| 항목 | 내용 |
|---|---|
| 대상 | 중학교 3학년 |
| 대단원 | 통계 - 대푯값 |
| 차시·시간 | 2차시, 차시당 45분(총 90분) |
| 수업 모형 | CBL(Challenge Based Learning): Engage - Investigate - Act, 모둠 협력 학습 |
| 핵심역량 | 문제 해결, 정보 처리, 추론, 의사소통, 공동체 역량 |
| 모둠 구성 | 4~5명, 기록자·계산 검증자·기기 담당·발표자 역할 분담 |
| 준비물 | 교사용 PC(Node.js 18 이상), 같은 와이파이에 연결된 학생 기기, 앱 접속 QR/URL, 프로젝터, 계산기, 모둠 활동지, 필기구 |

> **수업의 핵심 원칙:** 앱의 자동 추천은 정답이 아니라 판단을 돕는 하나의 근거이다. 학생은 원자료, 정렬 결과, 극단값과 반복값, 자료를 대표하려는 목적을 함께 검토하여 대푯값을 비판적으로 선택한다.

## 2. 도전 과제와 CBL 구성

| CBL 요소 | 수업 설계 |
|---|---|
| Big Idea | 생활 자료를 공정하고 설득력 있게 요약하기 |
| Essential Question | 하나의 값이 여러 사람의 생활 자료를 어떻게 공정하게 대표할 수 있을까? |
| Challenge | **우리 반의 생활 자료를 가장 공정하고 설득력 있게 대표하는 값을 찾아 근거를 제시하라.** |
| Guiding Questions | 평균·중앙값·최빈값은 어떻게 구하는가? 극단값이나 반복값은 각 대푯값에 어떤 영향을 주는가? 숫자 자료와 문자 자료는 어떻게 대표해야 하는가? 앱의 추천과 우리의 판단이 다르면 무엇을 확인해야 하는가? 선택한 값으로 어떤 실천을 제안할 수 있는가? |
| Guiding Activities | 사례 비교, 손계산과 정렬, 대푯값 선택 토론, 앱 자료 입력·수정, 자동 산출 결과 검증, 자동 추천 비판, 대푯값 탐정 게임, 근거 기반 제안 발표 |
| Guiding Resources | 교과서, 활동지의 정렬표·간단한 점그림, 계산기, 대푯값 수업 웹앱(`/teacher`, `/student`), 교사가 준비한 비식별 생활 자료 |
| Engage | 1차시: 극단값이 포함된 평균 용돈 사례로 ‘공정한 대표’의 의미를 문제화한다. |
| Investigate | 1차시: 세 대푯값의 계산과 선택 기준을 탐구한다. 2차시: 우리 반 자료를 앱으로 수집하고 원자료와 자동 산출·추천을 검증한다. |
| Act | 2차시: 조별 대푯값을 결정하고 탐정 게임으로 추론을 검증한 뒤, 자료에 근거한 생활 제안을 공유한다. |

## 3. 차시별 학습목표와 성취기준 연계

특정 성취기준 번호는 제시하지 않는다. 본 수업은 2022 개정 수학과 교육과정이 강조하는 **실생활 맥락의 자료 수집·정리·해석, 적절한 대푯값의 선택, 디지털 도구를 활용한 탐구, 근거에 기반한 비판적 의사소통과 협력적 문제 해결**의 취지를 반영한다.

| 차시 | 학습목표 | 교육과정 취지와의 연계 |
|---|---|---|
| 1차시 | ① 평균·중앙값·최빈값의 뜻을 설명하고 계산한다. ② 극단값·반복값·자료의 목적에 따라 적절한 대푯값을 선택하고 근거를 말한다. | 자료를 수학적으로 요약하고, 동일한 자료도 목적과 분포에 따라 다르게 해석될 수 있음을 추론·의사소통한다. |
| 2차시 | ① 디지털 도구로 생활 자료를 입력·수정하고 자동 산출값을 원자료로 검증한다. ② 적절한 대푯값을 근거와 함께 선택하여 결과를 해석하고 실천 제안을 만든다. | 디지털 소양을 바탕으로 자료를 비판적으로 분석하며, 협력적으로 의사 결정하고 실생활 문제 해결에 활용한다. |

## 4. 1차시 교수·학습 과정안(Engage 및 Investigate 중심)

| 단계 | 시간 | CBL 단계 | 교수 활동 | 학생 활동 | 자료 및 유의점 | 과정중심평가 |
|---|---:|---|---|---|---|---|
| 도입 | 7분 | Engage | 전체 도전 과제를 제시한다. “일주일 용돈이 3,000, 3,500, 4,000, 4,000, 4,500, 5,000, 25,000원일 때 ‘보통 7,000원’이라는 말은 공정한가?”라고 묻는다. | 직관적으로 대표값을 예상하고, 25,000원이 판단에 미치는 영향을 짝과 이야기한다. | 개인을 평가하거나 실제 가정 형편을 묻지 않는 가상 자료임을 밝힌다. | 선개념과 ‘공정함’의 판단 근거를 진단한다. |
| 개념·계산 탐구 | 10분 | Investigate | 평균, 중앙값, 최빈값의 뜻과 계산 절차를 정리한다. 자료를 반드시 정렬한 뒤 중앙값을 찾게 한다. | 합계와 자료 수를 확인해 세 값을 계산하고 서로 검산한다. | 합계 49,000원, 자료 수 7명, **평균 7,000원**, 정렬했을 때 네 번째 값인 **중앙값 4,000원**, 두 번 나타나는 **최빈값 4,000원**. | 계산 과정에서 합계, 나눗수, 정렬, 빈도를 정확히 확인하는지 관찰한다. |
| 자료 특성 비교 | 15분 | Investigate | 25,000원을 5,500원으로 바꾼 자료와 비교하게 한다. “극단값을 바꾸면 어느 값이 가장 크게 변하는가?”, “같은 값이 뚜렷이 반복되면 무엇이 유용한가?”를 발문한다. | 활동지에 정렬표 또는 간단한 점그림을 그리고 변화 전후를 비교한다. 바꾼 자료의 평균·중앙값·최빈값을 구한다. | 바꾼 자료: 3,000, 3,500, 4,000, 4,000, 4,500, 5,000, 5,500원. 합계 29,500원, **평균 약 4,214.29원**, **중앙값 4,000원**, **최빈값 4,000원**. 앱에 그래프 기능이 없으므로 점그림은 활동지에 직접 그린다. | 극단값에 대한 평균의 민감성과 중앙값의 안정성을 비교 설명하는지 확인한다. |
| 선택 기준 토론 | 8분 | Investigate | 상황 카드(소득처럼 치우친 자료, 시험 점수처럼 비교적 고른 자료, 선호 메뉴처럼 범주형 자료)를 주고 대푯값을 선택하게 한다. | 모둠별로 ‘값-자료 특성-목적’의 연결을 근거로 선택하고 반례를 검토한다. | 기준: 평균은 모든 값을 반영하나 극단값에 민감함, 중앙값은 정렬상 가운데로 극단값에 비교적 안정적임, 최빈값은 가장 잦은 값·범주를 보여 주나 없거나 여러 개일 수 있음. | 선택 자체보다 원자료 특성과 목적을 근거로 설명하는지 평가한다. |
| 정리 | 5분 | Engage→Investigate 연결 | “내일 앱이 추천한 값이 우리 판단과 다르면 무엇을 먼저 확인할까?”를 묻고 다음 차시를 예고한다. | 개인 문장으로 ‘대푯값 선택 규칙’을 쓰고 한 명씩 공유한다. | ‘평균이 항상 가장 정확하다’, ‘중앙값은 자료의 중간 크기다’, ‘최빈값은 반드시 하나다’라는 오개념을 즉시 교정한다. | 출구 문장에서 계산법과 선택 기준을 구분하는지 확인한다. |
| **합계** | **45분** |  |  |  |  |  |

### 1차시 핵심 선택 기준과 오개념 지도

- **평균:** 모든 관측값을 반영하므로 자료가 비교적 고르고 전체 수준을 비교할 때 유용하다. 단, 큰 극단값 하나가 평균을 끌어올릴 수 있다.
- **중앙값:** 자료를 크기순으로 정렬했을 때 가운데 값이다. 자료 수가 짝수이면 가운데 두 값의 평균이며, ‘최댓값과 최솟값의 중간’이 아니다.
- **최빈값:** 가장 자주 나타난 값으로 문자형 자료에도 사용할 수 있다. 최빈값이 없거나 여러 개일 수 있으며, 가장 큰 값이라는 뜻이 아니다.
- 어느 대푯값도 언제나 최선은 아니다. 분포, 자료 유형, 대표하려는 목적을 함께 밝힌 답을 인정한다.

## 5. 2차시 교수·학습 과정안(Investigate 및 Act 중심)

| 흐름 | 시간 | CBL 단계 | 교사 활동 및 앱 조작 | 학생 활동 | 자료 및 유의점 | 과정중심평가 |
|---|---:|---|---|---|---|---|
| 도입 | 5분 | Engage→Investigate | 도전 과제와 선택 기준을 회상시킨다. `/teacher` 화면에서 반과 주제를 확인하고 학생용 QR/URL을 제시한다. “추천을 보기 전에 어떤 자료 특성을 찾을까?”라고 묻는다. | 극단값, 반복값, 자료 유형, 목적을 답하고 역할을 정한다. | 교사 PC와 학생 기기를 같은 와이파이에 연결한다. | 전시 학습 회상과 역할 참여를 확인한다. |
| 앱 접속·자료 입력 | 8분 | Investigate | 접속이 어려운 학생을 지원하고 제출 현황을 확인한다. 잘못 입력한 사례를 일부러 보여 주며 수정 방법을 안내한다. | `/student`에서 **반→조→번호·이름** 순으로 입장하고 주제별 값을 제출한다. 단위·범위를 대조하여 오류가 있으면 값을 수정한다. | QR 또는 `http://교사PC_IP:3000/student`. 개인정보 대신 번호·별칭 사용을 권장한다. | 자료 수가 인원과 맞는지, 단위·범위·중복·누락을 점검하는지 본다. |
| 분석·가설 | 10분 | Investigate | `/teacher`에서 숫자형의 평균·중앙값·최빈값·자료 수, 문자형의 최빈값·빈도를 보여 준다. 자동 추천과 이유를 공개한 뒤 “추천이 타당하려면 원자료에서 무엇이 보여야 하는가?”라고 묻는다. | 자기 조 원자료를 정렬표나 활동지 점그림으로 나타내고 손계산·어림으로 앱 결과를 검증한다. 자동 추천에 동의/반대하고 조별 대푯값 가설을 정한다. | 앱은 그래프를 제공하지 않는다. 필요 시 활동지에 별도 시각화를 한다. 문자형에는 평균·중앙값을 적용하지 않는다. | 앱 값을 그대로 옮기지 않고 원자료, 극단값, 반복값, 목적을 근거로 검증하는지 평가한다. |
| 대푯값 탐정 게임 | 10분 | Investigate→Act | 교사 화면에서 게임을 시작하고 입력 자료가 있는 주제를 랜덤으로 뽑는다. 90초 동안 진행 상황을 관리한 뒤 정답을 공개한다. | 랜덤 주제에 대해 **자기 조 자료 중 실제 대푯값에 가장 가까울 것으로 예상되는 대표 학생**을 제안하고 설명한다. 조원 과반이 동의하여 확정한다. 공개 후 실제 대푯값과 대표 학생 값의 차이, 순위 점수(**+3/+2/+1**)를 확인한다. | 과반 확정이 필요함을 강조한다. 점수보다 추론 과정과 오차 원인을 중시한다. 최빈값일 때에는 앱의 빈도 순위 방식도 결과 화면과 함께 확인한다. | 제한 시간 내 합의, 대표 선택 근거, 결과에 따른 가설 수정 태도를 관찰한다. |
| 해석·제안 | 8분 | Act | “이 값이 우리 반 전체를 어떻게 설명하며 무엇은 숨기는가?”, “자료로부터 무리 없이 제안할 수 있는 행동은 무엇인가?”라고 묻는다. | 앱 결과와 게임 피드백을 활동지에 기록하고 최종 대푯값을 결정한다. 한계와 실천 가능한 제안을 한 문장씩 작성한다. | 인과관계를 단정하지 않고, 자료가 보여 주는 범위 안에서 제안한다. | 해석-근거-제안이 논리적으로 이어지는지 평가한다. |
| 공유·정리 | 4분 | Act | 2개 모둠의 30초 발표를 듣고 서로 다른 대푯값 선택도 근거가 타당하면 인정한다. 출구 티켓을 안내한다. | “선택한 값-근거-추천 검토-제안” 순으로 발표하고 출구 티켓을 작성한다. | 자동 추천과 최종 선택이 달라도 감점하지 않는다. | 근거 기반 의사소통과 협업을 종합 확인한다. |
| **합계** | **45분** |  |  |  |  |  |

## 6. 앱 입력 권장 주제

| 주제 | 유형·단위 | 권장 대푯값 | 수업 의도 | 입력 범위·규칙 |
|---|---|---|---|---|
| 어젯밤 수면 시간 | 숫자형, 시간 | 자동(평균/중앙값 비교) | 극단적으로 짧거나 긴 값의 영향 탐색 | 0~15, 0.5시간 단위; 특정 날짜 하루만 입력 |
| 어제 스마트폰 사용 시간 | 숫자형, 분 | 자동(중앙값 가능) | 치우친 분포와 자기관리 제안 | 0~1,000, 기기 사용 시간에서 총 분으로 확인 |
| 편도 통학 시간 | 숫자형, 분 | 자동(중앙값 가능) | 소수의 장거리 통학이 평균에 미치는 영향 | 0~180, 평소 집에서 학교까지 편도 기준 |
| 지난 7일 운동 횟수 | 숫자형, 회 | 자동(최빈값 가능) | 반복값의 의미와 범위가 작은 이산 자료 탐색 | 0~14, 20분 이상 활동을 1회로 통일 |
| 선호 급식 메뉴 유형 | 문자형, 없음 | 최빈값 | 범주형 자료에는 최빈값이 적절함을 확인 | 밥류/면류/국·찌개류/빵류/기타 중 하나만 입력 |
| 평일 아침 식사 횟수 | 숫자형, 일 | 자동 | 0~5의 생활 습관 자료를 해석하고 실천 제안 | 최근 평일 5일 기준, 0~5의 정수 |

민감한 건강 상태, 가정 형편, 성적, 몸무게 등은 주제로 사용하지 않는다. 응답이 불편한 주제에는 ‘응답하지 않음’을 허용하되, 숫자 칸에 임의의 0을 넣지 않도록 안내한다.

## 8. 평가 계획

### 4수준 분석 루브릭

| 평가 요소 | 4수준: 매우 충실 | 3수준: 충실 | 2수준: 부분적 | 1수준: 도움이 필요 |
|---|---|---|---|---|
| 개념·계산 | 세 대푯값의 뜻을 구분하고 정렬·합계·빈도를 정확히 검산한다. | 뜻과 계산이 대체로 정확하며 작은 오류를 스스로 고친다. | 일부 개념을 혼동하거나 계산 과정의 검증이 부족하다. | 계산 절차와 용어를 지속적으로 혼동한다. |
| 자료 해석 및 선택 | 분포, 극단값, 반복값, 자료 유형과 목적을 종합해 선택하고 한계까지 밝힌다. | 주요 자료 특성과 목적을 근거로 적절히 선택한다. | 특징을 언급하지만 선택과 근거의 연결이 약하다. | 자동 추천이나 직관만 제시하고 자료 근거가 없다. |
| 앱 활용·검증 | 정확히 입력·수정하며 자료 수와 원자료로 모든 산출·추천을 비판적으로 검증한다. | 입력 규칙을 지키고 주요 산출값을 확인한다. | 앱 사용은 가능하나 오류 점검이나 검증이 부분적이다. | 입력 오류를 방치하거나 앱 결과를 무검토로 수용한다. |
| 근거 기반 의사소통·협업 | 근거를 명료하게 말하고 반론을 반영하며 역할·과반 합의를 공정하게 수행한다. | 근거를 말하고 맡은 역할과 합의 과정에 참여한다. | 근거가 짧거나 참여가 고르지 않다. | 주장만 반복하거나 협업·합의에 거의 참여하지 않는다. |

점수 합산보다 각 요소에서 확인된 행동을 서술하고, 다음 탐구에서 개선할 한 가지를 제시한다.

## 9. 교사 사전 준비 체크리스트와 장애 대응 플랜

### 사전 준비 체크리스트

- [ ] 교사 PC에 Node.js 18 이상이 설치되어 있는지 확인한다.
- [ ] 프로젝트 폴더에서 최초 준비 시 `npm install`, 수업 전 `npm start`로 서버를 실행한다.
- [ ] `/teacher` 화면에 접속하여 반·조와 숫자형/문자형 주제를 만들고 단위를 확인한다.
- [ ] 서버에 표시되는 학생 URL을 QR로 준비하고, 교사 PC와 학생 기기가 같은 와이파이인지 확인한다.
- [ ] 학생 기기 2대 이상으로 `/student`에서 반→조→번호·이름 입장, 제출·수정까지 시험한다.
- [ ] 테스트 자료와 기존 개인정보의 정리 방법을 앱 제공 기능에서 확인한다. 필요한 자료는 안전하게 백업한 뒤 정리한다.
- [ ] 평문으로 저장되는 개인정보를 최소화하도록 실명 대신 별칭·번호 사용 원칙을 안내하고, 수업 후 자료 정리를 권고한다.
- [ ] `data/classes.json`은 함부로 수동 편집하거나 삭제하지 않는다. 앱이 제공하는 관리 기능과 안전한 백업·정리 절차를 먼저 확인한다.
- [ ] 인쇄 활동지, 계산기, 예비 핫스폿 사용 가능 여부와 시범용 자료를 준비한다.

### 예상 오개념과 피드백

| 예상 오개념/행동 | 교사 피드백 |
|---|---|
| 평균이 언제나 가장 정확하고 공정하다. | “25,000원을 5,500원으로 바꿀 때 평균은 얼마나 달라졌고 중앙값은 얼마나 달라졌는가?”로 반례를 확인시킨다. |
| 중앙값은 최댓값과 최솟값의 평균이다. | 원자료를 직접 정렬하고 자료 수가 홀수/짝수인 두 경우의 가운데 위치를 표시하게 한다. |
| 최빈값은 가장 큰 값이며 반드시 하나다. | 각 값 위에 빈도를 표시하고, 빈도가 모두 1인 자료와 공동 최빈값 자료를 비교한다. |

| 앱 추천이 곧 정답이다. | “추천 근거가 원자료에서 실제로 보이는가? 목적이 바뀌어도 같은 값을 고를까?”를 묻고 정렬표로 검증시킨다. |
| 자료 수가 달라도 결과를 그대로 비교한다. | 먼저 자료 수와 누락·중복을 확인한 후 비교하게 한다. |
| 대푯값만 알면 분포 전체를 안다. | 같은 대푯값을 가지지만 퍼짐이 다른 간단한 두 자료를 제시하고 원자료의 한계를 쓰게 한다. |
