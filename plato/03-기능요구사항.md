# PLATO 기능 요구사항

## 1. 구현 목표

PLATO의 1차 구현 목표는 학생이 `y = a(x-p)² + q` 형태의 이차함수를 직접 조작하며 그래프의 성질을 발견하도록 돕는 웹앱 MVP를 만드는 것입니다.

---

## 2. 사용자 역할

| 역할 | 설명 | 주요 권한 |
|:---|:---|:---|
| 학생 | 수업 중 웹앱으로 탐구 활동 수행 | 그래프 조작, 미션 수행, 설명 제출 |
| 교사 | 수업 진행 및 결과 확인 | 미션 설정, 학생 결과 확인, 데이터 다운로드 |
| 개발자/교사 제작자 | 앱 수정 및 배포 | 코드 수정, 미션 추가, 배포 관리 |

---

## 3. 핵심 기능 목록

### 3.1 그래프 렌더링
- 좌표평면 표시
- x축, y축, 눈금, 원점 표시
- 이차함수 그래프 실시간 표시
- 그래프 확대/축소는 MVP에서 선택 사항
- 그래프는 기본적으로 `x = -10`부터 `x = 10` 범위 표시

### 3.2 계수 조작
| 변수 | 기본값 | 범위 | 간격 | 설명 |
|:---:|:---:|:---:|:---:|:---|
| `a` | 1 | -5 ~ 5, 단 0 제외 권장 | 0.1 또는 0.5 | 방향과 폭 조절 |
| `p` | 0 | -5 ~ 5 | 0.5 또는 1 | 좌우 이동, 꼭짓점 x좌표 |
| `q` | 0 | -5 ~ 5 | 0.5 또는 1 | 상하 이동, 꼭짓점 y좌표 |

필수 조건:
- 슬라이더 조작 즉시 그래프가 업데이트되어야 한다.
- 현재 식이 수학적으로 읽기 쉬운 형태로 표시되어야 한다.
- `a = 0`은 이차함수가 아니므로 선택 불가하거나 경고 표시한다.

### 3.3 수학 정보 표시
앱은 다음 정보를 자동으로 계산해 표시합니다.

| 정보 | 표시 예시 |
|:---|:---|
| 현재 식 | `y = 2(x - 1)² - 3` |
| 꼭짓점 | `(1, -3)` |
| 축의 방정식 | `x = 1` |
| 방향 | 위로 열림 / 아래로 열림 |
| 폭 | 기본보다 넓음 / 기본 / 기본보다 좁음 |
| y절편 | `a(0-p)² + q` 계산값 |

### 3.4 탐구 질문
각 조작 단계에서 학생에게 질문을 제시합니다.

예시:
- `q` 값을 0에서 3으로 바꿔 보세요. 그래프는 어떻게 변했나요?
- `p` 값을 0에서 2로 바꿔 보세요. 꼭짓점은 어디로 이동했나요?
- `a` 값을 1에서 -1로 바꿔 보세요. 어떤 점이 달라졌나요?
- `a` 값을 0.5, 1, 2로 바꿔 보세요. 폭은 어떻게 변하나요?

### 3.5 미션 모드
미션은 조건, 목표, 성공 판정, 힌트로 구성됩니다.

```json
{
  "id": "mission-vertex-01",
  "title": "꼭짓점 이동하기",
  "condition": "꼭짓점이 (2, -3)인 포물선을 만들어라.",
  "success": {
    "p": 2,
    "q": -3
  },
  "hint": "꼭짓점은 (p, q)와 관련 있어요."
}
```

필수 미션 5개:
1. 꼭짓점 맞추기
2. 방향 맞추기
3. 폭 맞추기
4. 꼭짓점과 방향을 동시에 맞추기
5. 목표 점 또는 목표 영역 통과하기

### 3.6 설명 입력
학생은 각 미션 또는 탐구 질문 후 자신의 설명을 작성합니다.

필드:
- 학생 코드
- 미션 ID
- 현재 `a`, `p`, `q`
- 학생 설명
- 제출 시각

### 3.7 데이터 저장
MVP에서는 서버 없이도 동작하도록 다음 방식 중 하나를 우선합니다.

| 방식 | 장점 | 단점 | 추천도 |
|:---|:---|:---|:---:|
| 브라우저 LocalStorage | 구현 쉬움 | 기기별 저장, 교사 수합 불편 | 중 |
| CSV 다운로드 | 수업 후 수합 가능 | 학생이 직접 제출해야 함 | 상 |
| Google Form 연동 | 수합 쉬움 | 외부 서비스 의존 | 중 |
| 서버 DB | 분석 강력 | 구현·개인정보 관리 부담 | 후순위 |

추천 MVP:
- 학생 기기에서 활동 종료 시 `result.json` 또는 `result.csv` 다운로드
- 또는 교사가 QR 링크로 제출 폼 제공

## 4. 화면 요구사항

### 4.1 메인 탐구 화면
- 왼쪽: 조작 패널
- 오른쪽: 그래프 영역
- 하단: 질문과 설명 입력

### 4.2 미션 화면
- 현재 미션 조건 크게 표시
- 성공 여부 시각 표시
- 힌트 버튼 제공
- 미션 완료 후 다음 미션 이동

### 4.3 결과 화면
- 완료한 미션 목록
- 내가 만든 대표 그래프 1개
- 내가 발견한 규칙 요약
- 데이터 다운로드 버튼

### 4.4 교사용 화면, 후순위
- 반 전체 완료율
- 미션별 정답률
- 학생별 설명문 보기
- CSV 통합 다운로드

## 5. UX 원칙

1. **조작은 즉시 반응해야 한다.**  
   슬라이더를 움직이면 그래프가 끊김 없이 바뀌어야 합니다.

2. **수학 정보는 부담스럽지 않게 보여준다.**  
   초반에는 식, 꼭짓점, 축 정도만 표시하고 심화 정보는 접을 수 있게 합니다.

3. **정답보다 탐구 과정을 강조한다.**  
   미션 성공만 보여주지 않고 “어떤 값을 어떻게 바꿨는지”를 묻습니다.

4. **실패를 학습 경험으로 만든다.**  
   틀렸을 때 정답을 바로 주지 않고 조작할 변수에 대한 힌트를 제공합니다.

5. **모바일과 태블릿을 고려한다.**  
   슬라이더 터치 영역을 충분히 크게 만들고, 작은 화면에서는 세로 배치로 전환합니다.

## 6. 기술 스택 후보

### 6.1 단순 구현안
- HTML
- CSS
- JavaScript
- SVG 또는 Canvas

장점:
- 배포가 쉽고 의존성이 적음
- GitHub Pages에 바로 배포 가능
- 교사가 수정하기 쉬움

### 6.2 확장 구현안
- React 또는 Svelte
- SVG/Canvas 그래프 컴포넌트
- 상태 관리
- CSV/JSON 내보내기

장점:
- 컴포넌트화 쉬움
- 미션 추가와 대시보드 확장에 유리

### 6.3 추천
MVP는 **정적 HTML/CSS/JS + SVG**로 시작하고, 기능이 커지면 React/Svelte로 전환합니다.

## 8. 접근성 요구사항

- 색만으로 방향과 성공 여부를 구분하지 않는다.
- 그래프 주요 정보는 텍스트로도 제공한다.
- 버튼과 슬라이더에 명확한 라벨을 붙인다.
- 키보드 조작을 지원한다.
- 작은 화면에서도 글자가 너무 작아지지 않게 한다.

---

## 9. 개발 단계

| 단계 | 목표 | 산출물 |
|:---:|:---|:---|
| 1단계 | 기본 그래프와 슬라이더 구현 | 작동하는 탐구 화면 |
| 2단계 | 수학 정보 표시 | 식, 꼭짓점, 축, 방향, 폭 표시 |
| 3단계 | 미션 모드 구현 | 미션 5개, 성공 판정, 힌트 |
| 4단계 | 설명 입력과 데이터 저장 | CSV/JSON 다운로드 |
| 5단계 | 수업 적용용 polish | 반응형 UI, 오류 처리, 활동지 연동 |

---

## 10. 완료 정의

MVP가 완료되었다고 판단하는 기준:

- 학생이 별도 설치 없이 URL 접속만으로 사용할 수 있다.
- 5개 미션이 정상 작동한다.
- 그래프 조작 결과가 수학적으로 정확하다.
- 학생 설명과 미션 결과를 저장하거나 다운로드할 수 있다.
- 교사가 45~50분 수업에서 무리 없이 진행할 수 있다.
