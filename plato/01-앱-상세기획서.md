# PLATO 앱 상세 기획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|:---|:---|
| 프로젝트명 | PLATO: Parabola Learning And Thinking Observatory |
| 주제 | 이차함수 그래프 탐구 웹앱 |
| 대상 | 중학교 3학년 또는 고등학교 1학년 |
| 수업 차시 | 2차시, 각 45~50분 기준 |
| 핵심 개념 | 이차함수의 그래프, 포물선, 꼭짓점, 축, 평행이동, 폭과 방향, 절편 |

## 2. 교육적 문제 정의

### 2.1 학생들이 어려워하는 지점
1. **식과 그래프의 연결 부족**  
   `y = a(x-p)² + q`에서 `p`, `q`가 꼭짓점 좌표와 연결된다는 사실을 외우지만, 왜 그런지 직관적으로 이해하지 못합니다.

2. **계수 변화의 의미 혼동**  
   `a`의 부호와 크기가 그래프의 방향과 폭에 미치는 영향을 암기형으로 처리합니다.

3. **표현 간 전환의 어려움**  
   표, 식, 그래프, 언어 설명 사이를 자유롭게 오가지 못합니다.

4. **흥미 부족**  
   이차함수 그래프를 실제 세계의 움직임, 게임, 디자인과 연결하지 못하면 단순 공식 단원으로 받아들입니다.

### 2.2 PLATO가 해결하려는 문제
PLATO는 “설명을 듣고 따라 그리는” 수업에서 벗어나, 학생이 직접 그래프를 움직이며 규칙을 발견하도록 합니다. 특히 다음 경험을 제공합니다.

- 변수를 조작하면 그래프가 즉시 변하는 실시간 피드백
- 조건을 만족하는 그래프를 만드는 미션형 활동
- 자신의 조작 과정을 언어로 설명하는 메타인지 활동
- 교사가 학생별 탐구 과정을 확인할 수 있는 데이터 기록

## 3. 핵심 학습 목표

### 3.1 지식·이해
- 이차함수 `y = a(x-p)² + q`의 그래프가 포물선임을 이해한다.
- `a`의 부호와 절댓값이 그래프의 방향과 폭에 미치는 영향을 설명한다.
- `(p, q)`가 꼭짓점 좌표이고 `x = p`가 축의 방정식임을 설명한다.
- 이차함수의 그래프를 보고 식의 일부를 추론할 수 있다.

### 3.2 과정·기능
- 슬라이더 조작을 통해 그래프 변화의 규칙을 발견한다.
- 조건에 맞는 이차함수 그래프를 구성한다.
- 식, 그래프, 언어 설명을 연결하여 자신의 생각을 표현한다.
- 친구의 그래프와 자신의 그래프를 비교하며 수학적으로 토론한다.

### 3.3 정의적 목표
- 이차함수 그래프를 조작 가능한 대상으로 인식한다.
- 수학적 시행착오를 긍정적으로 경험한다.
- 게임형 미션을 통해 개념 탐구에 흥미를 느낀다.

### 3.4 디지털·AI 역량
- 웹 기반 수학 도구를 목적에 맞게 활용한다.
- 디지털 조작 결과를 근거로 수학적 설명을 작성한다.
- 선택적으로 AI 피드백을 활용해 자신의 설명을 개선한다.

## 4. 사용자 페르소나

### 4.1 학생 A: 공식은 외우지만 그래프가 약한 학생
- 특징: 문제 풀이 공식은 암기하지만 그래프 변화 설명이 약함
- 필요: 조작과 시각화를 통한 직관 형성
- PLATO 경험: 슬라이더를 움직이며 “아, q가 올라가면 그래프 전체가 위로 가는구나”를 발견

### 4.2 학생 B: 수학 흥미가 낮은 학생
- 특징: 추상적인 식을 지루해함
- 필요: 짧고 명확한 성공 경험, 게임형 미션
- PLATO 경험: “목표 지점을 통과하는 포물선 만들기” 미션으로 몰입

### 4.3 학생 C: 심화 탐구가 가능한 학생
- 특징: 빠르게 규칙을 발견하고 확장 문제를 원함
- 필요: 현실 모델링, 도전 과제
- PLATO 경험: 농구공 궤적을 포물선으로 모델링하고 최적의 슛 궤적 찾기

### 4.4 교사
- 필요: 학생이 어떤 조작을 했는지, 어디서 오래 머물렀는지 확인
- PLATO 경험: 학생별 미션 완료 여부, 조작 횟수, 설명문을 확인하고 후속 피드백 제공

## 6. 학습 흐름

### 6.1 1단계: 자유 탐색
- 학생은 `a`, `p`, `q` 슬라이더를 마음대로 움직입니다.
- 앱은 그래프와 함께 다음 정보를 실시간 표시합니다.
  - 현재 식: `y = a(x-p)² + q`
  - 꼭짓점: `(p, q)`
  - 축: `x = p`
  - 방향: 위로/아래로
  - 폭: 넓음/보통/좁음

### 6.2 2단계: 발견 질문
학생은 다음 질문에 답합니다.
- `q`만 바꾸면 그래프는 어떻게 움직이는가?
- `p`만 바꾸면 그래프는 어떻게 움직이는가?
- `a`가 음수가 되면 어떤 일이 생기는가?
- `|a|`가 커질수록 그래프는 어떻게 변하는가?

### 6.3 3단계: 미션 해결
예시 미션:
1. 꼭짓점이 `(2, -3)`인 포물선을 만들어라.
2. 아래로 열린 포물선을 만들어라.
3. 폭이 기본 그래프보다 좁고, 꼭짓점이 y축 오른쪽에 있는 그래프를 만들어라.
4. 주어진 두 점을 지나는 포물선을 찾아라.
5. 목표 상자 안에 꼭짓점을 넣고, 목표 점을 통과하게 만들어라.

### 6.4 4단계: 개념 정리
학생은 앱에서 제공하는 문장 틀을 활용해 규칙을 정리합니다.

예시:
```text
나는 q 값을 크게 하면 그래프가 _______ 방향으로 이동한다는 것을 발견했다.
p 값이 커지면 꼭짓점은 _______쪽으로 이동한다.
a 값이 음수이면 그래프는 _______ 열린다.
```

### 6.5 5단계: 확장 모델링
- 농구공 궤적 이미지를 배경에 두고 포물선을 맞춥니다.
- 학생은 다음을 설명합니다.
  - 꼭짓점은 실제 상황에서 무엇을 의미하는가?
  - `a` 값은 궤적의 어떤 성질과 관련 있는가?
  - 같은 출발점과 도착점을 지나는 여러 포물선 중 어떤 것이 자연스러운가?

## 7. 재미 요소 설계

### 7.1 게임형 미션 카드
각 미션은 짧고 명확하게 제시합니다.

| 난이도 | 미션 예시 | 성공 기준 |
|:---:|:---|:---|
| 쉬움 | 꼭짓점을 `(1, 2)`로 이동시키기 | 꼭짓점 좌표 일치 |
| 보통 | 아래로 열리고 폭이 좁은 그래프 만들기 | `a < 0`, `|a| > 1` |
| 어려움 | 두 목표 점을 모두 지나는 그래프 찾기 | 오차 범위 내 통과 |
| 도전 | 농구공이 림을 통과하도록 궤적 만들기 | 목표 영역 통과 |

### 7.2 즉각 피드백
- 성공 시: “조건을 만족했어요! 지금 그래프의 꼭짓점은 무엇인가요?”
- 실패 시: “꼭짓점은 맞았지만 방향이 달라요. a의 부호를 확인해보세요.”
- 힌트: 한 번에 정답을 주지 않고, 조작할 변수를 알려줍니다.

### 7.3 뱃지
- 꼭짓점 탐험가: 꼭짓점 미션 3개 완료
- 계수 마스터: `a` 변화 설명 완료
- 모델링 디자이너: 현실 궤적 미션 완료
- 설명왕: 친구가 이해할 수 있는 설명 작성

## 8. MVP 범위

### 반드시 구현할 기능

- `a`, `p`, `q` 슬라이더

- 반별 통계
- 모바일 최적화 고도화

## 9. 성공 기준

### 수업 성공 기준
- 학생 80% 이상이 `a`, `p`, `q`의 역할을 말로 설명할 수 있다.
- 학생 70% 이상이 조건에 맞는 포물선을 스스로 만들 수 있다.
- 학생들이 단순 정답보다 “어떤 값을 어떻게 바꿨는지”를 설명한다.

### 앱 성공 기준
- 첫 화면 진입 후 1분 이내 조작 방법을 이해할 수 있다.
- 슬라이더 조작에 대한 그래프 반응이 즉시 나타난다.
- 교사가 수업 후 학생별 활동 결과를 확인할 수 있다.
